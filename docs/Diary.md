## 2026-03-01 — YouTube: запасной вариант через Data API v3 при 404 RSS

RSS по `playlist_id=UU...` для канала BWF TV тоже возвращает 404. Добавлена поддержка **YouTube Data API v3**: при наличии в `.env` переменной **YOUTUBE_API_KEY** лента канала запрашивается через API (channels → contentDetails.relatedPlaylists.uploads, затем playlistItems). Без ключа по-прежнему используется RSS; при 404 в сообщении об ошибке предлагается добавить YOUTUBE_API_KEY. В config добавлено поле `youtube_api_key`; в docs/setup.md — раздел «YouTube: RSS 404» с инструкцией по получению ключа в Google Cloud Console.

## 2026-03-01 — Ротация логов

### Изменения

- **utils/logger.py:** вместо `logging.FileHandler` используются `RotatingFileHandler` для `bot.log` и `errors.log`. При достижении размера файла **2 МБ** создаётся новый файл, предыдущие переименовываются в `bot.log.1`, `bot.log.2`, … (и аналогично для `errors.log`). Хранится **5** архивных файлов (константы `LOG_MAX_BYTES`, `LOG_BACKUP_COUNT`). Итого в каталоге `logs/` — не более ~12 МБ для основного лога и до ~12 МБ для лога ошибок.

## 2026-03-01 — Автоподнятие бота после сбоя и падения сервера

### Изменения

- **Systemd (deploy/rzdbadminton-bot.service):** добавлены `StartLimitIntervalSec=300` и `StartLimitBurst=5` — не более 5 перезапусков за 5 минут, чтобы избежать бесконечного цикла при постоянной ошибке. Сохранены `Restart=on-failure` и `RestartSec=10` для автоперезапуска при падении; при `systemctl enable` бот поднимается после перезагрузки сервера.
- **bot.py:** при необработанном исключении вызывается `sys.exit(1)`, чтобы systemd интерпретировал завершение как сбой и выполнял перезапуск.
- **run-watchdog.sh:** скрипт для ручного запуска с циклом перезапуска (без systemd): после любого завершения процесса пауза 10 с и повторный запуск. Остановка — Ctrl+C.
- **Документация:** в [docs/setup.md](setup.md) добавлен подраздел «Автоподнятие после сбоя и падения сервера» с описанием поведения systemd и варианта с `run-watchdog.sh`.
