## 2026-02-27 — Рефакторинг отчётности: уход от шаблона Excel

### Наблюдения

- `excel_reporter.py` (660 строк) работал на базе шаблона `Бадминтон_шаблон.xlsx` через fuzzy-matching имён, копирование стилей и динамические колонки. Форматирование «ехало» из-за сложной логики копирования стилей openpyxl.

### Решения

- Полный рефакторинг: `excel_reporter.py` теперь генерирует Excel **с нуля** из плоских данных БД — без шаблонных файлов.
- Добавлена функция `get_monthly_attendance_records()` в `repositories.py`: возвращает плоскую таблицу `(user_id, display_name, full_name, poll_date)` за весь месяц.
- Отчёт по кнопке «Отчёт» стал **месячным pivot**: строки = участники, столбцы = даты тренировок (Пн/Ср), ячейки = 1/пусто, итого-строка.
- Удалены `record_attendance`, `reset_report_file` и весь код работы с шаблонами (≈500 строк).
- Обновлены `admin_helpers.py`, `commands.py`, `scheduler.py` — теперь передают monthly records вместо списка имён.
- Команда `/resetreport` сохранена, но выдаёт информационное сообщение о новом формате.

### Проблемы

- Шаблон `Doc/Бадминтон_шаблон.xlsx` больше не используется (можно оставить как архив).
