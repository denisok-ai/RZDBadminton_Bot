# Правила НФБР и файл .docx — использование в квизе

## Источник правил для квиза

В квизе используются **Правила Национальной Федерации Бадминтона России (НФБР)**, утверждённые приказом Минспорта РФ от 12.07.2021 № 546 и последующими изменениями.

- **Официальная страница «Правила игры» НФБР:** https://nfbr.ru/documents/rules  
- Ссылка на эту страницу подставляется в объяснение правильного ответа после каждого вопроса квиза.

---

## Анализ: использование файла pravila_06_11_2024.docx

**Вопрос:** можно ли брать текст для квиза из файла правил в формате .docx (например `pravila_06_11_2024.docx`)?

**Ответ: да.** Реализовано следующее.

### 1. Формат .docx

- Файл .docx — это ZIP-архив с XML-документами (Word Open XML).
- Текст из параграфов и таблиц можно извлекать **без ручного копирования** с помощью библиотеки **python-docx**.

### 2. Что сделано в проекте

- В зависимостях добавлен пакет **python-docx** (`requirements.txt`).
- В конфиге появился опциональный параметр **`rules_docx_file`** (путь к .docx).
- В `services/llm.py` добавлена функция **`_get_rules_context_from_docx()`**:
  - читает указанный .docx;
  - извлекает текст из всех параграфов (`document.paragraphs`);
  - возвращает не более 6000 символов (чтобы не перегружать контекст LLM);
  - при ошибке или отсутствии файла возвращает пустую строку, квиз продолжает работать без контекста из файла.

### 3. Как использовать ваш файл

1. Скопируйте `pravila_06_11_2024.docx` в каталог проекта, например:
   - `Doc/pravila_nfbr.docx`
2. В `.env` задайте путь к файлу (относительно рабочей директории бота):
   ```env
   RULES_DOCX_FILE=Doc/pravila_nfbr.docx
   ```
3. Перезапустите бота. При генерации квиза в промпт будет подставляться фрагмент текста из этого файла — формулировки вопросов и объяснений будут опираться на актуальную редакцию правил НФБР.

Путь в `.env` может быть и абсолютным (например, если файл лежит в другом месте).

### 4. Ограничения

- Учитываются только **параграфы** документа. Текст в таблицах и в колонтитулах текущей реализацией не извлекается (при необходимости это можно добавить).
- В промпт попадает только **первые 6000 символов** — обычно этого хватает на разделы об игре, счёте, площадке; при очень большом документе можно увеличить лимит в `llm.py` (`QUIZ_RULES_CONTEXT_MAX_CHARS`).
- Если `RULES_DOCX_FILE` не задан или файл отсутствует, квиз работает по-прежнему: LLM опирается на общие знания о правилах НФБР, а в ответе по-прежнему подставляется ссылка на https://nfbr.ru/documents/rules.

### 5. Итог

Файл **pravila_06_11_2024.docx** (и любой другой .docx с правилами НФБР) **можно использовать** как источник текста для квиза: положить в проект, прописать путь в `RULES_DOCX_FILE` и перезапустить бота.
