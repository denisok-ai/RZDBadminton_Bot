# Сборка и перенос на сервер

> Подготовка к развёртыванию. При старте бот сам проверяет зависимости и схему БД.

**Дата:** 2026-03

---

## Что выполняется при старте бота

1. **Проверка зависимостей** (`utils/startup.py`)
   - Проверяется наличие обязательных пакетов (aiogram, sqlalchemy, aiohttp, apscheduler, openpyxl, telethon и др.).
   - Если какого-то пакета нет, выполняется `pip install -r requirements.txt`, затем проверка повторяется.
   - При повторной неудаче бот завершается с сообщением об ошибке.

2. **Инициализация и миграции БД** (`database/__init__.py`)
   - Создаются все таблицы, если их ещё нет (`Base.metadata.create_all`).
   - Для SQLite проверяется наличие колонок в существующих таблицах (например, `quiz_records.correct_answer`, `quiz_records.explanation`). При отсутствии колонки добавляются автоматически.
   - Таблица `quiz_votes` и остальные создаются при первом запуске.

Отдельно запускать `scripts/migrate_quiz_records.py` не требуется — миграция выполняется при старте.

## Подготовка сборки к переносу

### 1. Локально (перед выкладкой)

```bash
# Из корня проекта
cd /home/denisok/projects/RZDBadminton_Bot   # или ваш путь

# Проверка зависимостей и тесты (по желанию)
pip install -r requirements.txt
python3 -c "from utils.startup import ensure_dependencies; ensure_dependencies()"
python3 -m pytest tests/ -q

# Создать архив без виртуального окружения и кэша (для копирования на сервер)
git archive -o RZDBadminton_Bot.tar.gz HEAD
# или
zip -r RZDBadminton_Bot.zip . -x ".venv/*" -x "__pycache__/*" -x "*.pyc" -x ".git/*" -x "data/backups/*"
```

### 2. Файл окружения

- Скопировать `.env.example` в `.env` и заполнить переменные (см. [setup.md](setup.md)).
- На сервер переносить только `.env` (в репозиторий не коммитить).

## Перенос на сервер и запуск

### Вариант A: Docker (рекомендуется)

1. Скопировать проект на сервер (git clone или scp архива):

   ```bash
   ssh root@46.19.68.241
   cd /opt
   git clone <url_репозитория> RZDBadminton_Bot
   # или: scp RZDBadminton_Bot.tar.gz root@46.19.68.241:/opt/ && на сервере: cd /opt && tar -xzf RZDBadminton_Bot.tar.gz -C RZDBadminton_Bot
   ```

2. Создать `.env` в каталоге проекта:

   ```bash
   cd /opt/RZDBadminton_Bot
   cp .env.example .env
   nano .env   # заполнить BOT_TOKEN, ADMIN_ID, MAIN_CHAT_ID и остальное
   ```

3. Создать каталоги для данных и логов (если ещё нет):

   ```bash
   mkdir -p data logs
   ```

4. Сборка и запуск:

   ```bash
   cd /opt/RZDBadminton_Bot
   docker compose up -d --build
   ```

   При первом запуске в контейнере выполнятся проверка зависимостей (в образе уже установлены) и миграции БД. Логи: `docker compose logs -f bot` или `tail -f logs/bot.log`.

### Вариант B: Без Docker (systemd / ручной запуск)

1. На сервере установить Python 3.11+, скопировать проект в `/opt/RZDBadminton_Bot`.

2. Создать виртуальное окружение и зависимости:

   ```bash
   cd /opt/RZDBadminton_Bot
   python3 -m venv .venv
   source .venv/bin/activate
   pip install -r requirements.txt
   ```

3. Создать `.env`, каталоги `data`, `logs`.

4. Запуск (при старте бот сам доустановит недостающие пакеты, если запускать не из venv, и применит миграции БД):

   ```bash
   python3 bot.py
   ```

   Для автозапуска использовать systemd: см. [setup.md](setup.md), раздел «Systemd».

## Скрипт обновления на сервере

В проекте есть скрипт `deploy/update_on_server.sh`. На сервере можно выполнять:

```bash
cd /opt/RZDBadminton_Bot
chmod +x deploy/update_on_server.sh
./deploy/update_on_server.sh
```

Скрипт подтягивает код (git pull), пересобирает образ и перезапускает контейнер. Перед первым запуском убедитесь, что в каталоге есть `.env`.
